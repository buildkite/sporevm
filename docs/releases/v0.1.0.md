# SporeVM v0.1.0

Date: 2026-06-20

v0.1.0 is the first tagged SporeVM baseline. It marks a validated pre-1.0
aarch64 fork/fan-out release, not a compatibility promise for future spore
formats or manifests.

## Scope

- One fixed aarch64 board across HVF on Apple Silicon and KVM on Linux/aarch64.
- One-shot run, capture, resume, fork, and fan-out over diskless and
  immutable-rootfs spores.
- Content-addressed bundle pack, unpack, push, and pull paths for RAM chunks,
  rootfs artifacts, and sealed writable rootfs disk layers.
- Named lifecycle create, exec, list, and remove on HVF and KVM.
- Repeatable A1/KVM release validation through
  `mise run validate:release-a1-kvm`.

## Release Gate

Cut this release only from `main` after:

```console
mise run check
mise run validate:release-a1-kvm -- \
  --region REGION \
  --source-instance SOURCE_INSTANCE \
  --dest-instance DEST_INSTANCE \
  --bucket BUCKET \
  --source-peer-ip SOURCE_PRIVATE_IP
```

The A1/KVM gate covers direct-S3 diskless bundle pull, direct-S3 and HTTP-peer
chunked-rootfs bundle pulls, destination cache reuse, rootfs CAS cache reuse,
corrupt bundle/rootfs rejection, KVM networking smokes, and KVM resume on the
destination host.

## Known Limits

- Spore manifests remain pre-1.0 and may break between tags.
- Cross-backend restore remains diagnostic, not the release-critical product
  path.
- General block-device state is still out of scope. Rootfs-bound writable state
  is represented as sealed disk layers.
- Disk-backed named lifecycle suspend/resume remains follow-up work.
